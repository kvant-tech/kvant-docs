# Настройка и запуск

## Основные возможности раздела

- **Просмотр списка** всех бизнес-процессов организации
- **Добавление новых** бизнес-процессов с указанием названия
- **Редактирование, копирование, удаление и запуск** бизнес-процессов
- **Управление статусом** бизнес-процессов (активный/неактивный)
- **Просмотр истории запусков** и детализации по каждой задаче

## Работа с бизнес-процессами

### Создание и редактирование бизнес-процесса

Чтобы добавить новый бизнес-процесс, нажмите кнопку **Добавить**.
В появившемся окне введите название процесса и нажмите **Сохранить**.

![](/from-carrot-images/bp1.png)

После этого откроется схема, где можно построить сценарий бизнес-процесса.

Для редактирования существующего процесса нажмите на иконку с тремя точками рядом с его названием, чтобы получить доступ к меню: запустить, отредактировать, просмотреть запуски, создать копию или удалить.

![](/from-carrot-images/bp2.png)

### Управление статусом

**Переключатель статуса** управляет возможностью запуска бизнес-процесса.

- **В положении Активный** — бизнес-процесс будет выполняться
- **В положении Неактивный** — новые запуски не создаются, но текущие выполнятся до завершения

## Настройка триггера запуска бизнес-процесса

Триггер задаёт событие, при котором запускается бизнес-процесс. Доступны четыре типа триггеров:

### Типы триггеров

**Окончание вводного обучения** — процесс запускается, когда сотрудник завершает вводное обучение.
**Начало рабочей смены у сотрудников** — процесс запускается в момент начала рабочей смены у выбранного сотрудника на функции.
**Периодичный запуск** — процесс запускается с заданной периодичностью.
**Ручной запуск** — процесс запускается вручную по команде пользователя.

![](/from-carrot-images/bp3.png)

> **Примечание:** При изменении триггера в блоке Старт можно редактировать параметры запуска.


## Создание сценария бизнес-процесса

### Основные элементы бизнес-процесса

**Коммуникация** — задача или сообщение, которое создаётся в рамках процесса.

**Задержка** — пауза в выполнении процесса, позволяющая системе ждать наступления определённых условий.

**Условие** — проверка, которая определяет дальнейшее направление процесса (например, одобрена ли задача).

### Последовательность создания

1. **После кнопки Старт** создайте элемент Коммуникация с необходимыми полями (название, описание, крайний срок, отправитель, исполнитель и др.)

2. **Добавьте элемент Задержка**, чтобы задать интервал ожидания перед проверкой условия

3. **Создайте Условие**, которое будет проверять выполнение или одобрение задачи

### Соединение элементов

Элементы соединяются стрелками в последовательности:
**Старт → Коммуникация → Задержка → Условие**

У элемента «Условие» есть два выхода:

- **Зеленая стрелка** — условие выполнено, процесс идёт дальше
- **Красная стрелка** — условие не выполнено, процесс возвращается к «Задержке» для повторной проверки

## Важные рекомендации и особенности

### Ключевые моменты

**Блок «Задержка» перед условием «Одобрена ли задача»** — необходим для ожидания решения отправителя, иначе процесс может завершиться преждевременно.

**Зацикливание через блок «Задержка»** — для организации повторных проверок условия используйте возврат стрелки через блок «Задержка».

**Несколько входящих стрелок в один блок** — может привести к зависанию процесса, так как блок ожидает выполнения всех входящих стрелок (исключение — блок «Задержка»).

**Изменение триггера** — после смены триггера необходимо проверить корректность всех настроек задач.

**Функция «Привести в порядок»** — автоматически выравнивает блоки бизнес-процесса для удобства визуального восприятия.

## Особенности работы с отправителями и исполнителями задач

### Варианты назначения

Отправитель и исполнитель могут быть:

- **Конкретным сотрудником**
- **Администратором функции**
- **Исполнителем функции**
- **Сотрудником, указанным во входном параметре** (для некоторых триггеров)

### Логика назначения

Если отправитель или исполнитель отсутствует или в отпуске, задача будет создана от исполнителя на самого себя.

В случае отсутствия сотрудников на функции появляется уведомление об этом.

## Дополнительные возможности

### Расширенные функции

**Передавать данные в следующую задачу** — добавляет ссылку на текущую коммуникацию в комментарий следующей.

**Запрос доказательств** — можно включить запрос подтверждающих выполнение задачи документов.

**Чек-листы** — добавление контрольных списков в задачи для упрощения контроля выполнения.

## Работа с параллельными задачами

Бизнес-процесс может содержать несколько параллельных задач, запускающихся одновременно.

Для каждой задачи создаётся своя последовательность элементов: **Коммуникация → Задержка → Условие**.

Можно создавать последовательности задач, объединять и параллелить их, формируя сложные сценарии.

## Условие «Своё условие»

Позволяет задать собственные критерии проверки задачи.

Можно создавать несколько вариантов условия с выбором результата исполнения задачи.

Исполнитель при закрытии задачи выбирает результат из предложенных вариантов, что запускает соответствующий сценарий.

## Элемент «Запуск бизнес-процесса»

Позволяет в рамках одного процесса запускать другой, ранее созданный бизнес-процесс.

Запускать можно только процессы с триггером «Ручной запуск».

При включённой опции «Передавать данные в следующую задачу» добавляется ссылка на коммуникацию из запускаемого процесса.

## Запуск бизнес-процесса вручную

Для запуска процесса с триггером «Ручной запуск»:

1. Перейдите в раздел «Бизнес-процессы»
2. Нажмите на иконку с тремя точками рядом с нужным процессом
3. Выберите **Ручной запуск**
