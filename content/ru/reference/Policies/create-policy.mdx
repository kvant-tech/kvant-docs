# Создание и редактирование регламента

В этой статье описано, как создать новый регламент, заполнить его содержание и отредактировать существующий.

## Создание регламента

Для создания регламента перейдите в раздел **Регламенты** и нажмите кнопку **«Добавить»** в правом верхнем углу.

### Основные поля

- **Название** — краткое и понятное название регламента (обязательное поле)
- **Тип** — выберите тип регламента из списка. Тип определяет, требуется ли регламенту проходить проверку и одобрение перед публикацией, или он публикуется автоматически
- **Серия** — при необходимости укажите серию для группировки связанных регламентов

### Содержание

Содержание регламента заполняется в текстовом редакторе. Доступны следующие возможности:

- **Форматирование текста** — заголовки (H1, H2, H3), жирный, курсив, зачёркнутый текст, код
- **Списки** — нумерованные и маркированные
- **Цитаты** — выделение важных фрагментов
- **Примечания** — специальные блоки для заметок и предупреждений
- **Таблицы** — создание таблиц с заголовками
- **Изображения** — загрузка и вставка изображений
- **Видео** — встраивание видео через iframe
- **Разделители** — горизонтальные линии для визуального разделения разделов

### Кому назначить

Укажите, кому будет распространён регламент:

- **Всем** — для всех сотрудников организации
- **Всем администраторам** — только для администраторов
- **Администраторам уровня** — для администраторов определённого уровня
- **Исполнителям типа** — для исполнителей определённого типа
- **Администратору функции** — для администратора конкретной функции
- **Исполнителям функции** — для исполнителей конкретной функции

При необходимости можно указать **исключения** — группы сотрудников, которые не должны получить регламент.

Также можно ограничить получателей по **местоположению** (в офисе / дистанционно).

### Ответственный за пересмотр

Вы можете назначить **ответственного** за пересмотр регламента и указать **плановую дату пересмотра**. Это помогает поддерживать регламенты в актуальном состоянии.

### Тестирование

При создании регламента можно сразу добавить тест. Подробнее о тестировании — в статье [Тестирование по регламентам](/ru/reference/Policies/policy-testing).

### Публичный доступ

Если у вас есть соответствующее разрешение, вы можете включить **публичный доступ** — регламент будет доступен по ссылке без авторизации в системе. Это удобно для внешних сотрудников или партнёров.

## Что происходит после сохранения

После сохранения регламент получает статус **«В разработке»** (черновик). Дальнейшая судьба зависит от типа регламента:

- Если тип подразумевает **автоматическую публикацию** — регламент сразу публикуется и становится доступен получателям
- Если тип подразумевает **проверку и одобрение** — регламент нужно отправить на проверку. Подробнее — в статье [Модерация регламентов](/ru/reference/Policies/policy-moderation)

## Редактирование регламента

### Когда можно редактировать

Регламент можно редактировать, если он находится в одном из следующих статусов:

- **В разработке** — автор может свободно редактировать
- **Ожидает проверки** / **Проверяется** — редактировать может сотрудник с разрешением «Проверка регламента»
- **Отправлен на одобрение** / **На одобрении** — редактировать может сотрудник с разрешением «Одобрение регламента»

Опубликованный регламент нельзя редактировать напрямую. Чтобы внести изменения, используйте функцию **«Предложить изменения»** — подробнее в статье [Модерация регламентов](/ru/reference/Policies/policy-moderation).

### Метки

К регламентам можно добавлять **метки** для удобной группировки и фильтрации. Метки отображаются на карточке регламента в списке.
