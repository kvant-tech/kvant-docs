# Повторяющиеся коммуникации

Повторяющиеся коммуникации — это задачи, которые автоматически создаются по расписанию. Они подходят для регулярных процессов: ежедневных отчётов, еженедельных проверок, ежемесячных задач и любых других действий, которые нужно выполнять с определённой периодичностью.

## Где найти повторяющиеся коммуникации

Повторяющиеся задачи создаются и управляются на странице функции в организационной структуре (ОФС). Перейдите на нужную функцию — повторяющиеся задачи привязаны именно к функциям, а не к конкретным сотрудникам.

Список всех повторяющихся задач компании также доступен в разделе **Отчёты → Список повторяющихся задач** (для пользователей с соответствующим разрешением).

## Создание повторяющейся задачи

Для создания повторяющейся задачи откройте карточку создания. Форма состоит из двух частей: настройки расписания (слева) и содержание задачи (справа).

### Кому

Выберите, кому будут назначаться создаваемые коммуникации:

- **На всех** — задача создаётся для всех сотрудников, соответствующих условию
- **На всех администраторов** — только для администраторов
- **На администраторов по уровню** — для администраторов определённого уровня
- **На исполнителей по типу** — для исполнителей определённого типа
- **Администратор функции** — для администратора конкретной функции
- **Исполнители функции** — для исполнителей конкретной функции

### Кроме

При необходимости можно исключить определённых сотрудников или группы из получателей задачи. Нажмите **«Кроме»** и выберите, кого нужно исключить.

### Где находится сотрудник

Можно ограничить создание задач в зависимости от местоположения сотрудника:

- **Неважно** — задача создаётся для всех (по умолчанию)
- **В офисе** — только для сотрудников, отмеченных как находящиеся в офисе
- **Дистанционно** — только для удалённых сотрудников

## Периодичность

Выберите, как часто должна создаваться задача:

### Каждые будни

Задача создаётся каждый рабочий день в указанное время. Можно добавить несколько временных слотов, нажав **«+»** — например, задача будет создаваться в 09:00 и в 17:00.

### Еженедельно

Задача создаётся в выбранный день недели в указанное время. Можно добавить несколько дней — например, каждый понедельник в 10:00 и каждую пятницу в 16:00.

### Ежемесячно

Задача создаётся в определённый день каждого месяца (или выбранных месяцев) в указанное время. Доступен выбор конкретного числа (1–31) или варианта **«Последний день месяца»**. Можно добавить несколько дат.

### Каждые несколько дней

Задача создаётся с заданным интервалом в днях, начиная с указанной даты. Например: каждые 3 дня, начиная с 1 января.

### Выбранные даты

Задача создаётся в конкретные даты, которые вы указываете вручную. Подходит для нерегулярных повторений. Для каждой даты можно указать своё время.

### Создавать в выходные и праздничные дни

По умолчанию этот переключатель **выключен** — если запланированная дата попадает на выходной или праздничный день, задача будет перенесена на ближайший рабочий день.

Если переключатель **включён**, задача будет создаваться в любой день, включая выходные и праздники.

## Крайний срок

Укажите, когда задача должна быть выполнена:

- **Через N минут/часов/дней/месяцев после создания** — крайний срок рассчитывается автоматически от момента создания каждой задачи
- **До определённого дня недели и времени** — фиксированный день и время для завершения

### Обязательно к сроку

Если этот переключатель включён, задача помечается как строго обязательная к выполнению в указанный срок.

## Содержание задачи

В правой части формы заполните содержание коммуникации:

- **Название** — краткое и понятное название задачи
- **Описание** — подробное описание того, что нужно сделать
- **Ожидаемый результат** — что должно быть итогом выполнения
- **Доказательства** — описание того, какие доказательства выполнения нужно предоставить
- **Чек-лист** — список пунктов, которые нужно выполнить в рамках задачи

## Дополнительные настройки

Раскройте блок **«Дополнительные настройки»** для доступа к расширенным параметрам.

### Создавать для

- **Создавать коммуникации на всех** — каждому получателю создаётся своя задача при каждом срабатывании
- **Создавать коммуникации по очереди** — задача создаётся поочерёдно: сначала одному сотруднику, в следующий раз — другому, и так по кругу

### Постановщик

Кто будет указан отправителем создаваемых коммуникаций:

- **Непосредственный руководитель** — руководитель исполнителя (по умолчанию)
- **Администратор функции** — администратор выбранной функции
- **Исполнитель функции** — конкретный исполнитель выбранной функции

### Постановщик в выходной или отпуск

Что делать, если постановщик в момент создания задачи находится на выходном или в отпуске. Доступные варианты зависят от настроек организации.

### Не требует одобрения отправителем

Если переключатель включён, задача автоматически закрывается после выполнения исполнителем — без необходимости одобрения постановщиком.

Если выключён (по умолчанию) — после выполнения задача переходит в статус ожидания одобрения постановщиком.

### Контроль результата отправителем

Отображается, только если задача требует одобрения. При включении постановщик должен будет при одобрении указать, достигнут ли ожидаемый результат.

### Часовой пояс

- **По часовому поясу сотрудника** — задача создаётся в указанное время по часовому поясу каждого получателя
- **По часовому поясу организации** — задача создаётся по единому часовому поясу компании

### Посменный график

Если в организации используются посменные графики, можно включить или выключить их учёт при создании коммуникаций.

### Наблюдатели

Добавьте сотрудников, которые будут наблюдателями во всех создаваемых коммуникациях.

## Управление повторяющимися задачами

### Пауза и возобновление

Вы можете приостановить повторяющуюся задачу — новые коммуникации перестанут создаваться. Для возобновления нажмите **«Запустить»**.

### Редактирование

Откройте повторяющуюся задачу для редактирования — изменения будут применены к будущим коммуникациям. Уже созданные задачи останутся без изменений.

### Копирование

Создаёт копию повторяющейся задачи с теми же настройками. Название копии будет начинаться со слова «Копия». Удобно для создания похожих задач с небольшими отличиями.

### Удаление и восстановление

При удалении повторяющейся задачи новые коммуникации перестанут создаваться. Удалённую задачу можно восстановить — после восстановления создание коммуникаций по расписанию возобновится.

## Часто задаваемые вопросы

### Как создать задачу, которая приходит каждый рабочий день?

Выберите периодичность **«Каждые будни»** и укажите нужное время. Убедитесь, что переключатель «Создавать в выходные и праздничные дни» выключен.

### Как создать задачу на 10-е число каждого месяца, но если это выходной — на ближайший рабочий день?

Выберите периодичность **«Ежемесячно»**, укажите 10-е число и нужное время. Переключатель «Создавать в выходные и праздничные дни» должен быть выключен — система автоматически перенесёт задачу на ближайший рабочий день.

### Как создать задачу на последний день месяца?

Выберите периодичность **«Ежемесячно»** и в выпадающем списке дня выберите **«Последний день месяца»**.

### Задача не создаётся — в чём может быть причина?

Проверьте:
- Не стоит ли задача на паузе
- Не попадает ли дата создания на выходной/праздник при выключенном переключателе «Создавать в выходные»
- Активен ли тариф организации
- Есть ли у функции назначенные сотрудники

### Можно ли создать задачу, которая приходит разным сотрудникам по очереди?

Да. В дополнительных настройках выберите **«Создавать коммуникации по очереди»** — система будет назначать задачу каждый раз следующему сотруднику из списка получателей.
