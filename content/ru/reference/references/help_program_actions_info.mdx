#### Поля проекта

**Название проекта:** Текстовое поле для ввода названия проекта.

**Описание:** Текстовое поле ввода для описания проекта.

**Ожидаемый конечный результат (ОКР)** В поле указывается ожидаемый конечный результат (ОКР) от выполнения проекта.

#### Дата начала и Дата окончания проекта

**Дата начала:** В поле указывается дата, когда проект официально стартует. Это может быть дата, когда начинаются первые действия по проекту, например, разработка плана, выделение ресурсов или начало работы над задачами.

**Дата окончания:** В данном поле указывается дата, когда проект официально завершится. Это может быть дата, когда все задачи проекта выполнены, достигнуты все цели и результаты переданы заинтересованным сторонам.

**Примечания:**

1. Проект будет отображаться на Таймлайне только в том случае, если заполнены оба поля: Дата начала проекта и Дата окончания проекта.

2. При создании проекта Дата начала и Окончания устанавливается по умолчанию на одну неделю вперед от текущей даты.

#### Ответственный

В поле Ответственный указывается сотрудник, который будет ответственным за выполнение проекта.

Если указан ответственный сотрудник, то отправителем задач проекта будет он. Если ответственный сотрудник не указан, то задача придет от сотрудника, который создал задачу.

**Примечание:** При разворачивание проекта из шаблона поле Ответственный является обязательным к заполнению

#### Дата начала отсчета крайнего срока

В данном поле указывается дата и время, от которой будет начинаться отсчет для Крайнего срока у задач проекта из шаблона.

**Примечание:** Данное поле отображается только при разворачивании проекта из шаблона и оно является обязательным к заполнению.

***Например:** Если в шаблоне добавлена задача с Крайним сроком +2 часа и при разворачивании проекта из шаблона в поле "Дата начала отсчета крайнего срока" указали дату и время "12.12.2024 12:00", то у задачи Крайний срок будет "12.12.2024 14:00".*

#### Общий доступ

В окне "Общий доступ" расположены настройки "Видимость проекта" и "Кто может редактировать проект".

![](/help_images_ru/share_ru.png)

**Видимость проекта:** у данной настройки есть несколько вариантов видимости проекта:

1. **Открытый** - проект будет отображаться в списке проектов у всех сотрудников организации.
2. **По ссылке** - проект будет отображаться в списке проектов только у тех сотрудников, у которых есть задачи в рамках этого проекта, также проект по ссылке может открыть любой сотрудник.
3. **Закрытый** - проект будет отображаться в списке проектов только у тех сотрудников, которые добавлены на ОФС проекта, а также у создателя и у ответственного проекта. По ссылке у других сотрудников данный проект не будет открываться.

**Кто может редактировать проект:** у данной настройки есть несколько вариантов кто может редактировать проект

1. **Ответственный сотрудник** - редактировать проект и добавлять/редактировать задачи в проекте может ответственный сотрудник проекта и создатель проекта
2. **Ответственный сотрудник и участники проекта** - редактировать проект и добавлять/редактировать задачи в проекте может ответственный сотрудник проекта, создатель проекта и также участники проекта
3. **Любой** - редактировать проект и добавлять/редактировать задачи в проекте может любой сотрудник организации

#### Файлы

В блоке с файлами отображаются все файлы, которые были добавлены в сообщения и комментарии в коммуникациях данного проекта.

![](/help_images_ru/projects_files_ru.png)

#### Коммуникации

Данный блок используется для работы со списком задач проекта - их добавления, редактирования и удаления.

Добавить задачи можно несколькими способами:

1. Вверху блока нажать на иконку "+" - после нажатия на кнопку в списке задач появится новая строка вверху таблицы, в которой необходимо заполнить нужные поля;
2. С помощью кнопки "Новая коммуникация" - после нажатия на кнопку в списке задач появится новая строка внизу таблицы, в которой необходимо заполнить нужные поля;

![](/help_images_ru/block_communications_ru.png)

Задача создается в случае, если заполнены обязательные поля Название и Исполнитель. Если данные поля не заполнены, то создается заготовка задачи, которая сохраняется в списке задач проекта, но пока не отображается ни у кого в разделе Коммуникации.

Просмотреть и редактировать созданную задачу можно с помощью нажатия на иконку ![](/help_images_ru/icon_open_eye.png), просмотреть заготовку нельзя, у заготовки отображается иконка ![](/help_images_ru/icon_closed_eye.png), удалить задачу или заготовку из проекта можно по нажатию на иконку ![](/help_images_ru/icon_delete.png).

***Для горизонтального перемещения по блоку с коммуникациями используйте комбинацию shift + колесико мыши, либо shift + ←/→.***

**Примечания:**

1. Отправителем задачи является ответственный сотрудник (если указан в свойствах проекта) или сотрудник, который создал задачу.

2. В участники коммуникации нельзя добавить ответственного за проект сотрудника - он не будет отображаться в списке доступных для добавления в участники сотрудников.

3. Если у задачи просрочен Крайний срок, то сменить исполнителя нельзя.

**Фильтр коммуникаций**

Фильтруются коммуникаций по статусу. Всего есть 6 статусов:

1. Входящие
2. Принято
3. В работе
4. Готово
5. Выполнено
6. Запланировано

#### Функциональная схема

В данном разделе можно построить и управлять организационной функциональной схемой проекта. Организационная функциональная схема (ОФС) - это схема, на которой отображены все функции в проекте и подчиненность между ними.

Для того, чтобы построить ОФС, нажмите на "Добавить функцию".

![](/help_images_ru/add_function_ru.png)

При наведении на функцию появляется иконка, позволяющая открыть дополнительные элементы управления функцией - переместить, добавить подфункцию, редактировать функцию, удалить функцию, управление функцией, и сортировка, также вызвать данное меню можно нажатием на функцию правой кнопкой мыши.

![](/help_images_ru/function_control_unit.png)

Нажатие на кнопку "Переместить функцию" позволяет перенести отдельную функцию или все подразделение в другое место ОФС.

Далее в окне отобразится функция, которую Вы перемещаете (1), а также необходимо будет выбрать функцию, под которую нужно переместить функцию (2).

![](/help_images_ru/function_migration.png)

Так выбранная функция переместиться под другую функцию и будет у нее в подчинении.

**Примечание:** Перемещать можно только под административную функцию (т.е. под ту, у которой нет исполнителей)

Если у нескольких функций один руководитель, то Вы можете создать "Административную функцию". То есть одну административную над ними функцию.

Тогда функции, которые будут ей подчинены, будут называться техническими. Техническая функция - функция, которая непосредственно производит необходимый результат. В то время как административная функция лишь управляет с помощью определенных административных распоряжений.

У административной функции не может быть исполнителей, так как исполнителями для административной функции, являются те исполнители, которые указаны в технических функциях.

![](/help_images_ru/adm_tech_function.png)

#### Повторяющиеся задачи

Данный раздел позволяет создавать и управлять повторяющимися задачами (ПЗ). ПЗ - это такой тип задач, которые выполняются с определенной периодичностью. Например:

- Каждый день
- Каждую среду
- Каждое 5 число месяца

Повторяющиеся задачи будут создаваться автоматически, согласно той периодичности, которую Вы настроите.

Для того, чтобы добавить повторяющуюся задачу, нажмите на кнопку "Добавить".

С созданной повторяющейся задачей можно выполнить следующие действия:

- поставить на паузу (1), тогда новые коммуникации не будут создаваться до тех пор пока Вы вновь не запустите её. Для запуска ПЗ нажмите на значок ![](/help_images_ru/repeat_task_play.png) у повторяющейся задачи;
- просмотреть список созданных коммуникаций по данной ПЗ (2);
- редактировать (3);
- скопировать (4) - при нажатии на кнопку откроется точная копия ПЗ, в самом начале названия задачи будет слово "Копия";
- удалить (5).

![](/help_images_ru/rt_icons.png)

#### Бизнес-процессы

В данном разделе вы можете добавить уже существующий бизнес-процесс в проект. Для этого нажмите на кнопку **Добавить** и выберите нужный бизнес-процесс из списка.

При добавлении бизнес-процесса вы можете изменить его триггер.

**Примечание:** нельзя изменить триггер у бизнес-процессов с триггерами "Окончание вводного обучения пользователя" и "Начало рабочей смены у сотрудников".

У бизнес-процессов с данными триггерами есть возможность менять только на следующие:

- **"Не выбрано"** - "Не выбрано", "Периодичный запуск", "Ручной запуск"; - **"Периодичный запуск"** - "Не выбрано", "Периодичный запуск", "Ручной запуск"; - **"Ручной запуск"** - "Не выбрано", "Периодичный запуск", "Ручной запуск".

#### Связанные проекты

В данном разделе отображаются связанные проекты.

Для того чтобы связать проекты необходимо нажать кнопку "Добавить", и в появившемся списке выбрать нужное - создать новый проекты (Новый проект) или выбрать уже имеющийся (Выбрать из списка).

![](/help_images_ru/bound_projects_ru.png)

При нажатии на "Новый проект" создание нового проекта будет открыто в новой вкладке. После того как проект будет создан, он будет отображен во вкладке "Связанные проекты" у проекта, из которого его создали, а также внутри нового проекта во вкладке "Связанные проекты" будет отображен тот проект, из которого его создали.

При нажатии на "Выбрать из списка" будет открыт список доступных вам проектов. После выбора нужного проекта он добавится в список связанных проектов.

Для того, чтобы удалить связь между проектами необходимо нажать на иконку с тремя точками, которая появляется при наведении на карточку проекта и выбрать единственное доступное действие - "Удалить". При удалении будет удалена связь, а не сам проект. Проект удалится из списка связанных проектов, но не удалится полностью из системы.

![](/help_images_ru/delete_bound_projects_ru.png)

Добавлять и удалять связь между проектами может любой сотрудник, у которого есть доступ к этим проектам.

**Примечание:** Отображение папки Связанных проектов включается в Настройках организации/Расширенные/Проекты.

#### Создание проекта из шаблона

Чтобы создать проект из шаблона, нужно в форме создания проекта кликнуть левой кнопкой мыши на поле *Название* (2) и в выпадающем списке выбрать нужный шаблон (1), либо начать вводить название нужного шаблона и выбрать нужный из результатов поиска.

![](/help_images_ru/title_project_ru.png)
